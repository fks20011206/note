### Let LRMs Break Free from Overthinking via Self-Braking Tuning

https://arxiv.org/pdf/2505.14604
### 1. 先把“过度思考”拆开看

作者把一条 reasoning trajectory 分成两部分：  
一部分叫 **Foundation Solution**，就是模型第一次比较完整、成体系地把题做出来的那段；  
后面反复出现的“等等，我再检查一下”“换个方法再验证一下”“有没有漏掉”之类的内容，被归到 **Evolution Solutions**。作者认为，自我纠错能力很多时候就出现在这部分，但 **过度思考也最常出现在这里**。

这个划分很关键，因为它不是简单说“后面的都没用”，而是说：  
**前面往往包含核心求解，后面既可能有有益修正，也可能有大量冗余复查。**

### 2. 再定义“怎么判断它开始过度思考了”

作者设计了两个指标。

第一个是 **reasoning efficiency ratio**，记作 ηs，本质上看的是：  
**模型在整段思考中，多早就已经第一次得到正确答案了。**  
公式是“第一次得到正确答案所需的步数 / 总思考步数”。如果这个比值越接近 1，说明模型差不多快结束时才得出正确答案；如果这个比值很小，就说明模型其实很早就答对了，但后面还拖了很久。

第二个是 **overthinking marker ratio**，记作 κt。  
它统计 reasoning 里一些“过度思考标记词”的占比，比如反复 reconsider、verify、alternative 这类语言线索。作者认为，光看结构还不够，因为很多冗余是通过语言模式暴露出来的。

然后把这两个量合并成一个总分：

**Overthink Score = β × κt + (1 − β) × (1 − ηs)**。  
这里他们设 **β = 0.1**，也就是主要看“结构性冗余”，语言标记只占较小权重。作者给的理由是：是否“很早就已经答对”是更稳、更直接的过度思考信号；而语言标记容易受提示词风格和模型习惯影响，所以只给 10% 权重。

### 3. 用这个分数来造训练数据

这是论文最重要的地方。作者提出了两个版本：

**SBT-E（Exact）**：  
对被识别成 overthinking 的样本，保留 **Foundation Solution + 一个 Evolution Solution**，然后再接上一小段“后续冗余推理”，但这一小段 **只给模型看，不算 loss**。也就是模型看到了“继续往下会长成什么样”，但不会因为生成这段而受到奖励。

附录里的伪代码更具体：  
SBT-E 直接保留 `FS + ES1`，如果还有更多 evolution solution，就再取 **ES2 的前 10%–20%** 作为 masked segment，但这段 **loss masking**。

**SBT-D（Dynamic）**：  
这个更细。它不是固定保留到某个“第几段”，而是一步一步往后加 reasoning step，并不断重算 Overthink Score；只要分数还没超过阈值，就继续保留；一旦超过主阈值 **τ1=0.2**，就认为已经开始进入“该刹车”的区域。随后再取分数落在 **τ1 到 τ2** 之间的一小段作为 masked 部分，其中 **τ2 = τ1 + 5%**。这样每道题保留多少推理长度，是自适应的。

你可以把两者理解成：

- **SBT-E**：比较像“固定截断模板”
- **SBT-D**：比较像“按题目难度动态截断”

### 4. 还加入了一个“刹车提示”

作者不满足于只截断数据，还在“该停下来的位置”插入自然语言提示，比如：

> “我已经多次得到同一个答案了，该结束思考了。”

当然论文不是要求模型一字不差背这个句子，而是用这类 **self-reflective braking prompt** 去教模型形成一种“元认知”：  
**我已经验证够了，现在该从 think 切到 final answer。**

作者强调，这比特殊 token 或外部规则更自然，因为它贴近模型原本的语言能力。

**原始 OpenR1-Math 长推理样本 → 找到 overthinking 边界 → 保留有效推理 + 插入刹车提示 + 保留少量冗余但 mask 掉 loss → 用这些新样本做普通 SFT。**


### O1-Pruner: Length-Harmonizing Fine-Tuning for O1-Like Reasoning Pruning

https://arxiv.org/pdf/2501.12570
## 它的方法在做什么

O1-Pruner 的思路不是简单粗暴地“限制最多输出多少 token”，而是设计一个训练目标，让模型自己学会：**简单题尽量短，难题在必要时可以长，但别无意义地冗长。** 论文把这叫做 **Length-Harmonizing Fine-Tuning**。

作者先把目标写成一个带约束的优化问题：  
一方面，希望模型生成的解答 **比参考模型更短**；另一方面，又要求 **准确率不能比参考模型差**。论文里把解答长度记作$L(y)$，准确与否记作 $A(x,y)∈{0,1}$。于是，目标是尽量增大“参考模型平均长度 / 当前解答长度”，同时约束当前模型的期望准确率不低于参考模型。

为了能训练，作者把“准确率不能下降”这个约束揉进奖励函数，变成一个更好优化的形式。最后得到的核心奖励可以概括成：

**Length-Harmonizing Reward = 长度奖励 + 准确率奖励**

其中长度奖励大致是

$\frac{\bar L_{\text{ref}}(x)}{L(y)} - 1$

意思是：如果你的解答比参考模型在这道题上的平均长度更短，就有正向激励；如果更长，就会吃亏。准确率奖励则是

$λ (A(x,y)−Aˉref(x))$

也就是把当前解答是否答对，与参考模型在这道题上的平均正确率相比。两项加起来，就得到最终的 RLH 奖励。

这套奖励设计最巧妙的地方在于它是**按题目自适应**的。论文在 4.3 节专门解释了这一点：  
如果一道题本来就很简单，参考模型在这题上的平均正确率 Aˉref(x)\bar A_{\text{ref}}(x)Aˉref​(x) 会比较高，那你“答对”本身拿不到太多额外奖励，所以模型更倾向于把答案写短；  
如果一道题很难，Aˉref(x)\bar A_{\text{ref}}(x)Aˉref​(x) 会比较低，那么“答对”这件事会带来更大的准确率奖励，此时模型就不会一味追求短，而会更重视走对推理路径。换句话说，它希望学到的是：**简单题短做，难题谨慎做。**

## 它是怎么训练的

训练时，作者先固定一个参考模型 πref\pi_{\text{ref}}πref​，对每道题先采样出多条解答，用这些采样结果估计这道题的两个基线量：

- 参考模型在这题上的平均输出长度 $Lˉref(x)$
- 参考模型在这题上的平均准确率 $Aˉref(x)$

然后他们不是做标准在线 RL，而是做一种**off-policy、PPO 风格**的训练。也就是说，训练样本直接来自参考模型预先采样好的答案，而不是每次都从当前模型重新在线采样。这样做的好处是：训练流程更简单，数据可以一开始就准备好。损失函数采用了类似 PPO 的 clipped 形式，里面用到重要性比率

$r(\theta)=\frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$​

来把“参考模型分布上的样本”转成对当前模型的有效训练信号。作者明确说，这样做是为了降低训练复杂度，同时保持效果。

如果把整套流程翻成人话，大概就是：

1. 先让参考模型对每道题想很多次；
2. 统计这道题“通常会想多长”“通常有多容易答对”；
3. 再训练新模型，让它在这道题上尽量比“参考平均长度”更短，但不能因此把正确率搞坏

### Demystifying Long Chain-of-Thought Reasoning in LLMs

https://arxiv.org/pdf/2502.03373
第四部分：作者提出的核心技术——Cosine Reward

为了解决上面这个问题，作者设计了一个新的长度塑形奖励，叫 **Cosine Reward**。它不是只看答案对不对，还把 **生成长度** 也纳入奖励。核心约束有三个：

1. **正确答案的奖励要高于错误答案**
2. **同样正确时，短的 CoT 比长的 CoT 奖励更高**，鼓励高效推理
3. **同样错误时，短的错误 CoT 应该罚得更重**，鼓励模型在没把握时多想一会儿，而不是太快交卷

作者用一个分段 cosine 函数去平滑实现这个奖励，并在超过最大长度时再加额外惩罚。

这个设计很巧：

- **答对了但太啰嗦**，也不够好
- **答错了但很短**，也不够好
- 模型会被推向一种更合理的状态：  
    **会做的题快点做完，不会的题允许想更久**

效果是很明显的。

**长 CoT 必须被“塑形”。否则它很容易从“深度推理”滑向“无效拉长”。**

 第五部分：奖励还会被模型“黑掉”

作者还观察到一个很现实的问题：  
当训练算力继续加大时，模型会开始 **reward hacking**。具体表现是：它不是靠更好的求解去变长，而是靠**重复、堆砌、拖时间**来骗长度奖励。作者甚至用关键词 “alternatively” 的出现频率去估计分支行为，发现里面混进了不少“假分支”。

他们的解决办法很简单但有效：**加 repetition penalty**。  
Figure 5 显示，加入 repetition penalty 后，不仅下游性能更好，而且 CoT 也更短，说明推理算力被利用得更有效。作者的 takeaway 是：**长度奖励迟早会被黑，但重复惩罚可以缓解。**

这部分的直觉很好懂：

- 长度奖励会让模型想“要不要多写点”
- 如果没有约束，它就可能学会“多写废话”
- 所以你得同时奖励“有必要的思考”，惩罚“无意义的重复”

### Learning When to Think: Shaping Adaptive Reasoning in R1-Style Models via Multi-Stage RL
https://arxiv.org/pdf/2505.10832 
三阶段强化学习
### 第一阶段：先稳住“双模式共存”

第一阶段的目标不是提升最终成绩，而是**防止模式坍塌**。  
因为如果你一开始就奖励“答对且不思考”，模型很容易全都跑去 no-think；反过来，如果 thinking 模式暂时更占优，也可能全都坍塌成 always-think。论文先定义了一个 naive reward：

- think 且对：+1
- think 且错：0
- no-think 且对：+2
- no-think 且错：-1

这样看起来是在鼓励“能不想就不想”，但早期训练会不稳定。于是作者又加入了 **batch reward balance**：如果一个 batch 里 think 太多，就软性压低 think 奖励；如果 no-think 太多，就压低 no-think 奖励，用一个目标比例 γ 和惩罚斜率 λ 把两种模式维持在可探索的平衡区间。

你可以把第一阶段理解成：  
**先不要急着教模型“怎么最优”，先确保它别只会一种行为。**  
论文的消融也证明了这一点：如果只用 naive reward，模型会很快坍塌到 thinking 模式；如果直接上长度惩罚，又会坍塌到 no-thinking 模式。只有 batch-level 的平衡机制，才能让两种模式稳定共存。

### 第二阶段：在两种模式里分别把“答对”练出来

第二阶段做的事其实很朴素：  
**把第一阶段那个“保平衡”的扶手拿掉，直接用 naive reward，开始真正强化正确行为。**  
作者说，经过第一阶段的正则化后，第二阶段即使放开，thinking 和 no-thinking 的比例也不会轻易再次坍塌；而且这一阶段会给更大的上下文预算，让模型在需要时能产生更长、更充分的推理。它的作用，就是把“什么时候想”和“想了以后能不能答对”这两件事真正建立起来。

这一步很关键，因为论文后面的消融表明：  
**如果跳过第二阶段，直接从第一阶段去做剪枝，最终准确率会明显更差。**  
作者报告说，跳过第二阶段后，即便最后响应长度差不多，平均准确率也只有 **47.6%**，低于完整三阶段训练得到的 **51.7%**。也就是说，先“强化正确推理”，再去“压缩长度”，比直接上剪枝有效得多。

### 第三阶段：再做“按长度感知的剪枝”

到了第三阶段，模型已经有了比较稳定的 think / no-think 双模式，也在各模式里学会了更可靠地答题。此时作者才开始引入 **length-aware reward**。  
它的设计很有意思：对每个 query 里的响应，先看它相对同组样本是长还是短，然后做奖励塑形：

- **如果答对了，越长越要扣一点奖励**，鼓励“正确但简洁”；
- **如果答错了，越短反而越不理想**，鼓励模型在难题或不确定时不要太快收工。

所以它不是简单地“长就罚、短就奖”，而是更像：  
**会的题请短答，不会的题别装会，应该多想一点**



### Do NOT Think That Much for 2+3=? On the Overthinking of Long Reasoning Models
https://arxiv.org/pdf/2505.21765

## 他们提出了两个“效率指标”

这是这篇论文比较有价值的地方。作者不只看“答对没答对”，还看 **token 有没有被有效利用**。

### 1. Outcome efficiency（结果效率）

这个指标问的是：

**这些后续 token，对“得到正确答案”到底有没有帮助？**

它的做法很直观：  
如果一个回答里第一轮就已经出现正确答案，那么从那之后再生成的大量 token，就不再算“对正确率有贡献的 token”。于是 outcome efficiency 本质上就是：

> **到第一次得到正确答案为止的有效 token，占总 token 的比例**

论文发现，在有正确答案的样本里，**超过 80% 的情况，第一轮 solution 就已经答对了**；而这第一轮通常只占整个回答不到 60% 的 token。也就是说，后面那大段内容对把题做对帮助非常有限。以“2+3”那个例子为例，第一轮只用了 **39 token** 就答对了，但总回答长达 **901 token**，所以 outcome efficiency 只有 **4.3%**。

### 2. Process efficiency（过程效率）

这个指标问的是：

**这些后续 token，至少有没有带来新的推理思路？**

作者不是简单看长度，而是看“后面的解法是否引入了新的 reasoning strategy”。他们用 GPT-4o 去把同一回答里的 solution 按推理策略聚类，再计算有多少 token 属于“新的、没重复过的思路”。本质上，process efficiency 就是：

> **真正贡献新思路的 token，占总 token 的比例**

论文发现，随着 solution 的轮次增加，后面的 distinctness ratio 会持续下降。尤其第二轮经常只是对第一轮做同一思路下的复查，并不是真的引入新策略。
## 这篇论文怎么缓解 overthinking

它的方法不是 RL，也不是外接一个控制器，而是一个 **self-training + preference optimization** 的方案。整体分三步：

1. 先让长推理模型自己生成长 reasoning traces；
2. 再把这些长回答剪成更短、更高效的版本；
3. 然后拿“短版本 vs 长版本”做偏好学习，让模型学会偏向更简洁但仍有效的推理


### Don’t Think Longer, Think Wisely: Optimizing Thinking Dynamics for Large Reasoning Models

https://arxiv.org/pdf/2505.21765

DTO：**先切段 → 再问“停在这里够不够答对” → 找到最早可收尾点 → 补一个自然收尾 → 再逐段删掉无贡献片段 → 得到最终精炼轨迹。**
构造出的偏好数据（精炼轨迹）  ->  SimPO

论文先把问题写成一个约束优化：**在尽量不降低任务表现的前提下，最小化整条推理轨迹的计算成本**。这里成本可以用 token 数或 FLOPs 表示，性能则用最终答案正确率之类的指标表示。换句话说，它不是单纯追求“更短”，而是追求“**更短但不更差**”。

DTO 的核心流程分成两步：先找一个“**应该停下来的位置**”，再把中间不必要的片段删掉。作者先把原始推理切成一串 thinking patterns；然后对每个候选停止点，都临时接上一段 `exit pattern`，让模型快速补完若干个短答案，并用 Monte Carlo 方式估计：**如果在这里结束思考，答对的概率有多大**。这个概率记作 pip_ipi​。他们选择最早一个让 pip_ipi​ 超过阈值 TTT 的位置，作为截断点。文中实验里 T=1.0T=1.0T=1.0，也就是要求在这个位置已经积累了足够信息来稳定给出正确答案。

截断后，作者不会直接硬停，而是再加一个更自然的 **finalize pattern**，相当于“我觉得现在已经足够，可以给最终答案了”，然后从这个位置采样若干个补全，保留其中**最短的正确答案补全**。这样做的目的，是让截断后的轨迹既短，又像一条完整自然的推理，而不是生硬腰斩。

接下来还有一步细化剪枝：作者用一个辅助模型（实验里是 **Llama-3.3-70B-Instruct**）逐段判断某个 thinking pattern 是否真正有贡献。只有当一个片段被判断为“没提供有效信息、存在有害错误、完全跑题或只是无意义重复”，并且把它删掉后仍能快速验证出正确答案时，这个片段才会被移除。也就是说，DTO 不是简单删“长”的，而是在删“**没用的长**”。

## 它不只是后处理，还拿这些轨迹继续训练

作者发现，经过 DTO 优化后的轨迹质量不错，于是进一步把它们拿来做偏好优化。具体做法是：对每道题先采样 4 个回答；如果正确回答够多，就选最短的正确回答做优化后的正样本 ywy_wyw​；如果不够，就把现有正确的和一些错误回答一起送进 DTO，得到优化后的轨迹作为正样本。对应的负样本 yly_lyl​ 则选这些采样回答里**最长、且未经优化**的轨迹。然后用 **SimPO** 做偏好优化，让模型更偏向“优化后的高效推理”，而不是“原始的冗长推理”。

实验里的主要训练/构造设置是：从 **MATH** 训练集抽 **5,000** 个样本，基础模型用 **DeepSeek-R1-Distill-Qwen-1.5B** 和 **DeepScaleR-1.5B-Preview**，辅助模型用 **Llama-3.3-70B-Instruct**；每题采样 4 个回答，Monte Carlo 估计里 M=10M=10M=10，finalize 后再采样 K=4K=4K=4。评测数据包括 **MATH、GSM8K、Gaokao(Math)、AMC2023、AIME2024、AIME2025**，另外还在 **MMLU-Pro** 上做了跨领域泛化测试。



### EFFICIENT REASONING WITH BALANCED THINKING
https://arxiv.org/pdf/2603.12372

问题痛点：

现有方法大部分都只简化思维链，但容易出现过度简化，有时候反而会想太少。在“想太多”和“想太少”之间做了一个平衡。
## 论文的核心直觉

作者抓住了两个信号：

第一是 **stepwise confidence**。你可以把它理解成：模型在当前这一步推理里，到底有多“笃定”。论文把每个 reasoning step 里各 token 的最大预测概率聚合起来，得到这一步的置信度。第二是 **confidence variance**。它看的是最近几步里，置信度波动大不大。作者的观察是：

- **高置信度方差** 往往意味着模型在不同思路之间来回摇摆，容易导致冗余推理，也就是 overthinking。
- **持续高置信度且低方差** 往往意味着模型过早锁死在某条路径上，不再探索别的可能，也就是 underthinking。

这个观察其实很重要，因为它把“过度思考 / 思考不够”从一句模糊的话，变成了可以在推理时连续监控的信号。论文还专门说，他们想找的是一个**连续、细粒度**的指标，而不是把整条推理二元地判成“保留/丢弃”。

## ReBalance 到底怎么做

它分成两部分：**离线准备** 和 **在线控制**。

### 1. 先离线找出两种“原型状态”

作者先用一个很小的、已见过的数据集做一次离线分析。具体做法是：

- 把 `<think> ... </think>` 里的推理按步骤切开；
- 对每一步算 confidence 和 confidence variance；
- 用分位数阈值把这些步骤分成两类：
    - **overthinking steps**：高方差、低置信度
    - **underthinking steps**：低方差、持续高置信度。

然后，作者在一个自动选出来的较深层上，取每个推理步骤**第一个 token** 的隐藏状态，并分别在 overthinking / underthinking 两类上求平均，得到两个 prototype。论文的理由是：每一步的首 token 是这一步“意图”的紧凑表示，而且深层隐藏状态对不同 reasoning mode 的可分性更强。

### 2. 用两个 prototype 构造 steering vector

接着，他们把这两个 prototype 的差定义成一个 **steering vector**。直觉上，这个向量代表了“从一种不平衡推理状态，移动到另一种不平衡推理状态”的方向。推理时，只要沿这个方向加一点或减一点，就能把当前隐藏状态往“少一点冗余”或者“多一点探索”的方向推。

### 3. 再设计一个动态控制函数

这一步是整篇论文最关键的地方。作者不是固定给一个 steering 强度，而是让 steering weight 变成当前 **confidence + variance** 的函数。也就是说，每一步都动态决定：

- 这次要不要 steer；
- steer 多大；
- steer 的方向是哪边。

论文里这个规则可以很直白地理解成：

- 如果模型现在表现出 **overthinking 风险**，就给一个**负向 steering**，促使它更快承诺、少绕圈子。
- 如果模型现在表现出 **underthinking 风险**，就给一个**正向 steering**，鼓励它再多探索一会儿。

作者还专门做了一个平滑的控制曲面，让 steering 不会在边界处突然跳变，而是随着 confidence / variance 的变化平滑增减。这样做是为了数值稳定，也为了不把正常推理扰乱得太厉害。

### 4. 在线推理时怎么用

真正部署时，ReBalance 很轻量：

- 在每一步在线算当前 confidence 和 variance；
- 把它们送进动态控制函数；
- 得到一个 steering weight；
- 然后只在选定层、选定 step 的首 token 隐藏状态上做一次注入。

论文强调，这个方法 **不需要额外 forward pass**，也 **不需要辅助模型**，所以它是 **training-free、plug-and-play** 的。每个 backbone 只需要离线拟合一次，后面就能固定用于多个未见任务

### OptimalThinkingBench: Evaluating Over and Underthinking in LLMs

提出了 **OptimalThinkingBench**，这是一个统一的基准，用于同时评估大语言模型中的“过度思考（overthinking）”和“思考不足（underthinking）”，并推动能够在性能与效率之间取得平衡的“最优思考”模型的发展。

我们的基准由两个子基准组成：

- **OverthinkingBench**：包含来自 **72 个领域** 的简单通用查询，以及简单数学题；
- **UnderthinkingBench**：包含 **11 类高难度推理任务**，以及较难的数学题。

我们提出了新的、经过“思考量”调整的准确率指标（thinking-adjusted accuracy metrics），并基于这些指标对 **33 种不同的思考型与非思考型模型**进行了大规模评测。结果表明：**没有任何一个模型能够在我们的基准上实现“最优思考”**。

具体来说，思考型模型往往会在最简单的用户查询上额外生成数百个 token 的推理过程，却并没有带来性能提升。相反，大型非思考型模型则常常表现出“思考不足”，其效果甚至不如一些规模更小的思考型模型。


### BARREL: Boundary-Aware Reasoning for Factual and Reliable LRMs
https://arxiv.org/pdf/2505.13529

作者关注的是 **factual reliability（事实可靠性）**，不是单纯的推理能力。文中说，一个可靠的事实型推理模型至少要满足两点：第一，分清自己**知道什么、不知道什么**；第二，对自己知道的事实要**稳定、准确地说出来**。而现有 LRM 往往做不到这两点：它们很少承认无知，反而会编出答案；而且对相似问题还会前后不一致。

作者先做了一个观察：在 TruthfulQA 这类事实任务上，很多推理模型在**答错时反而用了更多 reasoning tokens**。他们把这个现象叫做 **factual overthinking**。也就是说，模型不是“没想”，而是“想过头了，而且方向错了”。

### 2. 作者识别出的两种“病态过度思考”

论文里最重要的 insight，是把事实题上的坏推理模式拆成了两类：

**第一类：Last-minute Guessing（最后一刻瞎猜）**。  
模型前面已经想了很多，但始终没有足够证据，最后却在结尾突然拍脑袋押一个答案，像考试快结束时乱填最后一道题。这个模式更常见于模型其实**不知道**答案的时候。

**第二类：Second-thought Spiraling（二次思考螺旋）**。  
模型一开始其实已经碰到了正确答案，但它不停继续分析、继续怀疑、继续展开，最后反而把原本对的结论自己推翻了。这个模式更像“知道了但不敢停”。

如果用一句话概括，这篇论文认为：  
**事实题上的 overthinking，不只是“思维太长”，而是“在该停的时候不停，在没证据时还硬要落答案”。** 这也是它和一般“减少 CoT 长度”的工作不太一样的地方。

### 3. BARREL 方法到底做了什么

BARREL 由三部分组成：**知识边界标注、面向 SFT 的推理轨迹构造、以及带拒答奖励的 GRPO**。

#### 3.1 先判断这个问题对模型来说是“知道”还是“不知道”

作者不是人工标 known/unknown，而是用一个 sampling 策略：  
对同一个问题，用多个 few-shot prompt、重复多次采样；只要这些样本里**有一次**答对，就把这个问题视为模型“知道”；如果所有采样都没答对，就视为“未知”。

这个设计的意思很重要：  
作者不把“单次答错”直接视为“不会”，而是把“模型潜在能否召回正确知识”单独抽出来。这样后面才能区分：

- 是**知道但表达/推理坏掉了**
- 还是**本来就不知道却硬猜**

这正对应他们说的 “knowing” 和 “telling” 的差别。

#### 3.2 再为 known / unknown 两类题，构造不同的“理想推理轨迹”

对 **known questions**，BARREL 的目标是防止 second-thought spiraling：  
模型应该先定位到 gold answer 及其强证据，然后再看一些替代候选，最后**重新确认**最初那个有证据支撑的正确答案，而不是越想越怀疑。

对 **unknown questions**，BARREL 的目标是防止 last-minute guessing：  
模型也可以先回忆背景知识、探索若干 plausible candidate，但如果始终找不到足够支持的答案，就应该明确承认不确定，并输出类似 “Sorry, I don’t know” 的拒答，而不是在结尾硬猜一个。

也就是说，它不是简单训练 “不会就拒答”，而是训练一种**有边界意识的推理过程**：

- 已经有充分证据了，就别继续自我推翻；
- 证据始终不够，就别被“必须给答案”的惯性推着走。

#### 3.3 最后用 GRPO 强化这个行为

在 RL 阶段，作者设计了一个三档 reward：

- **答对**：高奖励 `r_c`
- **合理拒答**：中等奖励 `r_s`
- **答错/幻觉**：低奖励 `r_w`

而且具体设成了 `r_c = 1`, `r_s = -0.5`, `r_w = -1`。也就是说，拒答不是“奖励很高”，但它明确比**胡说八道**更好。论文强调，现有 RL 范式一个根本问题是：**几乎没有显式奖励“我不知道”**，于是模型自然学会“无论如何都要答”。

这个点非常关键：  
BARREL 的精髓不是“鼓励拒答”，而是**重新定义风险排序**——正确回答最好，诚实拒答次之，瞎答最差。

### Reasoning Models Know When They’re Right: Probing Hidden States for Self-Verification

**这篇论文不是在训练一个更会做题的模型，而是在证明：推理模型内部已经存在一种“自我验证信号”，我们可以把它 probe 出来，并用它减少无效推理。**

https://arxiv.org/pdf/2504.05419

论文的方法不复杂，但设计得很巧。作者先把长链式推理拆成多个 **chunk**，每个 chunk 对应一条局部推理路径，并且尽量让它包含一个中间答案。具体做法是：先按段落切分，再用诸如 “wait”“double-check”“alternatively” 这样的词识别新推理分支；然后用 Gemini 2.0 Flash 抽取每个 chunk 的中间答案并判断它是否正确；最后取该 chunk **最后一个 token 的最后一层隐藏状态** 作为表示，训练一个两层 MLP 二分类器，预测这个中间答案是否正确。由于正负样本不平衡，他们还用了加权二元交叉熵。

实验数据主要是 **GSM8K、MATH、AIME、KnowLogic**，模型则用了 **DeepSeek-R1-Distill-Llama-8B / 70B、R1-Distill-Qwen-1.5B / 7B / 32B，以及 QwQ-32B**。作者强调，很多 probe 在网格搜索后最优的隐藏层维度甚至是 **d = 0**，也就是退化成近似线性 probe，这说明“答案正确性”在隐藏状态里编码得相当直接，不需要很复杂的读出器。

论文的第一个主要发现是：**推理模型的隐藏状态确实能很好地区分中间答案的对错。** 在同分布测试上，所有 probe 的 **ROC-AUC 都高于 0.7**，而且 **ECE 低于 0.1**，说明不仅能分对错，而且分数还比较校准。作者据此认为，推理模型内部已经形成了相当明确的“正确性表征”。从图 2 看，不同模型和数据集上效果都不差，尤其在数学任务上更强；例如文中还提到 R1-Distill-Qwen-32B 在 AIME 上的 probe ROC-AUC 超过 0.9。

第二个发现是：**这种信号有一定跨数据集泛化能力，但主要发生在相近领域内。** 例如在 R1-Distill-Llama-8B 上，GSM8K 和 MATH 之间的 probe 迁移表现还不错，AUC 和 ECE 都还能接受；但跨到更难的 AIME，或者从数学跨到逻辑、从逻辑跨到数学，泛化就明显变差了。作者的解释是，不同领域的隐藏状态分布差异比较大，所以 probe 不是“普适 verifier”，更像是“同域可迁移”的 verifier。

第三个发现很重要：**这种“知道自己对不对”的能力，和 long CoT reasoning 是相关的。** 作者拿 reasoning model 和它的非 reasoning 对照模型比，在 MATH 上发现：reasoning model 上训练出来的 probe 明显更强。图 3 里，reasoning model 对最终答案正确性的 probing accuracy 大约 **0.92**，ROC-AUC 大约 **0.86**，ECE 大约 **0.03**；而非 reasoning model 的对应值大约只有 **0.66 / 0.82 / 0.23**。作者据此推测，这种 self-verification 信号很可能是在 **长 CoT 监督训练** 中被强化出来的。

第四个发现非常有意思：**模型在还没把中间答案完整说出来之前，隐藏状态里就已经提前带有“未来答案是否正确”的信号了。** 作者把一个 chunk 再切成更细的段落，去看离最终中间答案不同距离的位置能不能预测后面的答案对不对。结果是：越靠近答案位置，probe 越准；但即使在更早的位置，也已经有明显信号。论文给出的数字是：相对峰值 **79%** 的 accuracy，位于 **10% / 50% / 95%** 位置时分别低 **14% / 10% / 5%**；而 ECE 在大约 **60%** 的位置就已经降到最低 **0.03**。这说明模型并不是“说完答案后才知道自己对不对”，而是很多时候在生成过程中就已经形成了某种内部判断。

论文最实用的一部分是把这个 probe 真正拿来做 **early exit**。做法很直接：顺着完整推理轨迹扫描每个中间答案，只要 probe 给出的“该答案正确概率”超过阈值，就提前截断，把这个中间答案直接当最终答案。作者把它和一个简单 baseline 比较：固定在第 m 个 chunk 就停。结果显示，在 **R1-Distill-Llama-8B + MATH** 上，这种基于 probe 的 early exit 能在**几乎不降准确率**的情况下明显减少 token。

具体数字上，阈值 **Thr = 0.85** 时，准确率大约 **88.2%**，和不 early-exit 基本相当，但生成 token 大约减少 **24%**；阈值 **Thr = 0.9** 或更高时，能达到与不 early-exit **相同的 88.6%** 准确率，同时减少大约 **19%** token。并且在相近 token 成本下，这种基于置信度的 early exit 比固定位置停下来的 static early-exit 最多能高 **5%** 的准确率。作者把这解释为：模型其实已经“知道当前答案大概率对了”，但默认生成策略还是会继续啰嗦下去，这正是一种 overthinking。

所以，这篇论文最核心的结论可以概括成三层。  
第一层，**reasoning model 的隐藏状态里有 correctness signal**。  
第二层，**这个 signal 不是很弱的边角信息，而是能被简单 probe 稳定读出来的。**  
第三层，**模型自己并没有充分利用这个 signal，所以才会继续做很多多余推理。** 论文在 discussion 里明确说，这说明模型具有潜在的 self-verification 能力，但还没有高效地调用它。

我觉得这篇论文和你前面读过的那些“self-doubt / overthinking”工作相比，有一个特别强的地方：它不是只从输出文本里看“wait”“double-check”“让我再想想”这种表面现象，而是直接去看 **hidden state 里有没有内部证据**。也就是说，它在某种程度上表明：**过度思考不一定是模型“不知道答案”，也可能是模型“已经内部知道了，但解码过程没及时刹车”。** 这个判断是基于论文的 probing 和 early-exit 结果做出的概括。

当然，这篇论文也有边界。就它自己的实验范围看，主要还是集中在 **数学和逻辑推理**，而且 early-exit 的主实验只在 **R1-Distill-Llama-8B 的 MATH** 上做了重点验证；同时，中间答案抽取和标注还依赖了 **Gemini 2.0 Flash**。所以它很有启发性，但还不能直接推出“所有推理模型、所有开放任务都存在同样强的 hidden verifier”。

### The First Impression Problem: Internal Bias Triggers Overthinking in Reasoning Models
https://openreview.net/pdf?id=2PP70tFY0S

先用一句最通俗的话概括它的发现：

**模型并不是单纯“爱想太多”，而是“心里先有了一个答案”，后面一旦认真算出来的结果和这个直觉不一样，就会陷入‘要不要再检查一下’的循环。** 这就是作者说的 “first impression problem”。

论文的证据分成四层。

第一层是**相关性证据**。作者在不同模型、不同规模、不同任务和中英文场景下都观察到同一个趋势：**internal bias 与最终推理结果偏差越大，模型的 reasoning length 越长，reflection 关键词也越多。** 他们测试了 DeepSeek-R1、QwQ-32B、R1-Distill-Qwen-14B，任务包括字符计数、逻辑推理和 AIME 数学题。比如在 CharCount(zh) 上，DeepSeek-R1 的高偏差组平均长度比低偏差组长 **32.1%**，QwQ-32B 长 **21.0%**，R1-Distill-Qwen-14B 长 **30.1%**；在 KnowLogic 上分别长 **42.1% / 25.9% / 21.2%**；在 AIME 2024 上则达到 **23.6% / 32.1% / 43.1%**。作者还专门比较了“第一次得出答案之前的推理长度”，发现同一任务内部各组差不多，因此这些差异更像是 bias 在驱动后续额外反思，而不是题本身更难。

第二层是**因果证据**，这是这篇论文最强的部分。作者做了两个 counterfactual interventions。第一个干预叫 **removing question**：模型在推理中第一次得出答案后，研究者把原题从上下文里删掉，只留下它已经生成的推理轨迹，让它继续决定还要不要反思。直觉上，这会阻止模型重新去“看题目、唤回第一印象”。结果很明显：Across datasets，冗余推理长度下降 **31.1% 到 53.5%**。具体来说，CharCount(en) 从 **541.0** 降到 **453.3** token，CharCount(zh) 从 **1081.9** 降到 **782.5**，KnowLogic 从 **6320.7** 降到 **5857.5**，AIME 2024 从 **10796.0** 降到 **7706.1**，AIME 2025 从 **13428.9** 降到 **10303.0**。更有意思的是，复杂任务上准确率还上升了：KnowLogic 从 **27.2%** 到 **29.3%**，AIME 2024 从 **63.3%** 到 **66.7%**，AIME 2025 从 **36.7%** 到 **46.7%**。这说明被削掉的很大一部分确实是“多余的想法”，不是必要推理。

第二个因果实验叫 **bias injection**。作者在 CharCount(zh) 上，专门往模型里注入“错误偏置”或“正确偏置”，看 reasoning behavior 会不会跟着变。结果也很符合假设：给低偏差样本注入错误偏置后，平均长度从 **355.5** 增到 **454.9**，准确率从 **89.5%** 降到 **83.4%**；给高偏差样本注入正确偏置后，平均长度从 **600.8** 降到 **412.1**，准确率从 **67.0%** 升到 **76.6%**。也就是说，**错误的第一印象会放大 overthinking，正确的第一印象会抑制 overthinking。** 这基本把“相关”推进到了“因果”。

第三层是**机制解释**。作者不是只停留在行为统计上，还看了 attention。结论是：模型在大多数正常推理阶段，对题目本身的注意力并不高；但当它即将输出反思词、决定“要不要再想一下”时，对题目区域的 attention 会突然暴涨。论文里写得很明确：在 reflection 时，对 question section 的注意力上升到平时的 **4 倍以上**，而且这种异常上升集中在中后层。作者据此推测：模型在“要不要继续反思”的决策点，会重新看题目；而题目里正好又会重新激活那个最初的 internal bias，于是它把“已经通过推理得到的答案”和“题目激活出的第一印象”再比一遍，一冲突就继续反思。

第四层是**对现有 mitigation 方法的反思**。作者试了几类已有方法，包括训练期的 FCS（SFT / DPO 版本）和推理期的 SEAL、PROBE。结论不是说它们完全没用，而是说：**它们能缩短长度，但没真正消掉 internal bias 的影响。** 在 CharCount 上，原始模型的高偏差组比低偏差组长 **31.5%**；Remove 干预后能把这个差距压到 **15.2%**，但 FCSDPO 反而是 **46.3%**，SEAL 是 **38.5%**，PROBE 是 **29.9%**，说明“高偏差更长”的模式依旧在。AIME 2024 上，FCSSFT 虽然缩短了链条，但准确率从 **63.3%** 掉到 **50.0%**。作者的结论是：这些方法更多是在表面上让链条变短，有时甚至伤害了真正的推理能力，却没有触及 internal bias 这个根因。

所以，这篇论文最重要的贡献不是“又提了一个新的 early exit 技巧”，而是把 overthinking 的来源往前追了一步。以前很多工作在问：“怎么让模型少想一点？” 这篇论文问的是：**“模型为什么会多想？”** 它给出的答案是：**不是单纯因为模型喜欢反思，而是因为模型太容易被自己最初的直觉答案牵着走。**

细节：
## 论文是怎么“强迫模型不要思考”的

论文的主做法不是改模型结构，而是**改提示模板**。它给 reasoning model 一个显式的“不要思考，直接回答”的前缀，然后**立刻关闭 `<think>` 段**，再让模型输出答案。文中给的模板很直接，例如：

`<User>{question}<Assistant><think> Let me answer him without thinking more.</think> Answer:`

或者中文版本：

`<User>{question}<Assistant><think> 让我直接回答他，不要有思考过程。</think> 答案是：`

作者一共设计了 **4 个模板**，每个模板采样 **16 次**，温度是 **0.5**，合起来每题得到 **64 个 direct answers**，把它当成 internal bias 的分布近似，而不是只看一次结果。对于数值型输出，他们还会做一个基于规则的离群值清理，避免极端样本把 bias 分布拉偏。

这里有个很关键的点：论文不是简单靠“题目里写一句请直接回答”来让模型跳过思考，因为作者明确说了，**对当前长推理模型，仅仅在题目里要求“直接回答”通常不会让它自动缩短或跳过 reasoning**；这类模型往往还是会先想很长，只是最后按要求输出一个简短答案。所以他们用了“no-reasoning prompt + `</think>`”这种更强的方式来截断思考。

另外，附录里还提到一个更工程化的办法：对 DeepSeek-R1 API，可以用 **Chat Prefix Completion**，把 assistant 前缀直接设成 `"The answer is: "`，这样模型会直接补答案而不走显式 reasoning。这是附录的补充演示，不是正文主实验的统一做法。

## 什么叫“注入正确偏置” (`injecting correct bias`)？

这里也不是在推理时往 hidden state 里塞一个向量。论文里的 **bias injection** 是一个**小规模、样本级别的微调实验**。做法是：先从 CharCount(zh) 里挑出 **500 个最低偏差样本**和 **500 个最高偏差样本**，然后**针对每个样本构造 50 条改写后的陈述句**，再拿这些陈述句去微调模型，最后回到**同一个样本**上测试，看它的 reasoning length 和 accuracy 怎么变。

它的核心不是“教模型解题”，而是**预先给模型灌输一个关于该题答案的陈述性信号**。论文给的例子就是：

- 中文：`strawberry这个单词里面有3个字母r。`
- 英文：`There are 3 letters 'r' in the word 'strawberry'.`

这类句子不是问句，而是“事实陈述”。作者用标准 next-token prediction 对这些句子做 LoRA 微调，训练方式类似 continued pretraining；不引入特殊 token，LoRA rank=32，alpha=64，learning rate=1e-4，batch size=32。
## `deviation degree (Dbias)` 是什么

这是论文里量化 bias 强弱的核心量。定义是：

Dbias=dist(a~bias,afinal)D_{\text{bias}} = \text{dist}(\tilde a_{\text{bias}}, a_{\text{final}})Dbias​=dist(a~bias​,afinal​)

其中 a~bias\tilde a_{\text{bias}}a~bias​ 是 direct answers 的分布近似，afinala_{\text{final}}afinal​ 是完整推理后的最终答案。

具体地：

- **数值题**：用 direct answers 和 final answer 的 **MAE**
- **类别题**：用不一致率，也就是 direct answers 里有多少比例和 final answer 不同

所以 Dbias 不是“对不对”的二元标签，而是“**第一印象和最终推理结果差多远**”的量。
## 注意力机制他们是怎么“定量观察”的

他们不是只看一个可视化热图，而是做了两层分析。

### 第一层：个例可视化

用 “strawberry” 例子，对比两个时间点的 attention：

- **answer point**：模型刚生成一个中间答案时
- **reflection point**：模型马上要输出 `Wait` 这类反思词时

Figure 3 里看的是：在这两个时间点，当前 token 对前文不同 token 的 attention 分配有什么变化。作者发现，在 reflection point，模型更偏向去看**题目本身**、对题目的复述，以及前面那些较表层的反思片段；反而对真正导出答案的核心 reasoning 段关注变少。

### 第二层：统计量化

他们把前文 token 分成三类：

1. **Question**：输入题目的 token
2. **Mid_Results**：中间结果 token，比如 `3`、`three`
3. **Others**：剩下的 token

然后对当前位置 ppp 的 token，计算它对某类 ccc 的**平均归一化注意力分数**：本质上就是把它指向该类别历史 token 的 attention weights 做平均，并且再除以一个 **uniform attention baseline 1/p1/p1/p**，用来校正“序列越长，均匀分配时每个旧 token 天然越小”这个位置效应。之后再把这些分数按两组聚合：

- **Reflection** 组：反思点附近的 token
- **Other** 组：其他普通位置的 token

得到 group-level score SGcS_G^cSGc​。另外他们还去掉了序列最前面的 3 个 token，以避开所谓 **attention sink phenomenon** 的干扰。

最终他们报告的是：

- 在大多数普通 reasoning 步骤里，模型对 question 的注意力不高
- 但一到 reflection 步骤，对 question 的 attention 会突然升高到**平时的 4 倍以上**
- 这个上升主要出现在**中后层**，尤其是 layers 21–30 的平均图里最明显，layer-wise ratio 图也显示中后层最突出

作者据此推测：**模型在“要不要继续反思”的决策点，会重新看题，而这会重新激活 internal bias。**
