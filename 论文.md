### Let LRMs Break Free from Overthinking via Self-Braking Tuning

https://arxiv.org/pdf/2505.14604
### 1. 先把“过度思考”拆开看

作者把一条 reasoning trajectory 分成两部分：  
一部分叫 **Foundation Solution**，就是模型第一次比较完整、成体系地把题做出来的那段；  
后面反复出现的“等等，我再检查一下”“换个方法再验证一下”“有没有漏掉”之类的内容，被归到 **Evolution Solutions**。作者认为，自我纠错能力很多时候就出现在这部分，但 **过度思考也最常出现在这里**。

这个划分很关键，因为它不是简单说“后面的都没用”，而是说：  
**前面往往包含核心求解，后面既可能有有益修正，也可能有大量冗余复查。**

### 2. 再定义“怎么判断它开始过度思考了”

作者设计了两个指标。

第一个是 **reasoning efficiency ratio**，记作 ηs，本质上看的是：  
**模型在整段思考中，多早就已经第一次得到正确答案了。**  
公式是“第一次得到正确答案所需的步数 / 总思考步数”。如果这个比值越接近 1，说明模型差不多快结束时才得出正确答案；如果这个比值很小，就说明模型其实很早就答对了，但后面还拖了很久。

第二个是 **overthinking marker ratio**，记作 κt。  
它统计 reasoning 里一些“过度思考标记词”的占比，比如反复 reconsider、verify、alternative 这类语言线索。作者认为，光看结构还不够，因为很多冗余是通过语言模式暴露出来的。

然后把这两个量合并成一个总分：

**Overthink Score = β × κt + (1 − β) × (1 − ηs)**。  
这里他们设 **β = 0.1**，也就是主要看“结构性冗余”，语言标记只占较小权重。作者给的理由是：是否“很早就已经答对”是更稳、更直接的过度思考信号；而语言标记容易受提示词风格和模型习惯影响，所以只给 10% 权重。

### 3. 用这个分数来造训练数据

这是论文最重要的地方。作者提出了两个版本：

**SBT-E（Exact）**：  
对被识别成 overthinking 的样本，保留 **Foundation Solution + 一个 Evolution Solution**，然后再接上一小段“后续冗余推理”，但这一小段 **只给模型看，不算 loss**。也就是模型看到了“继续往下会长成什么样”，但不会因为生成这段而受到奖励。

附录里的伪代码更具体：  
SBT-E 直接保留 `FS + ES1`，如果还有更多 evolution solution，就再取 **ES2 的前 10%–20%** 作为 masked segment，但这段 **loss masking**。

**SBT-D（Dynamic）**：  
这个更细。它不是固定保留到某个“第几段”，而是一步一步往后加 reasoning step，并不断重算 Overthink Score；只要分数还没超过阈值，就继续保留；一旦超过主阈值 **τ1=0.2**，就认为已经开始进入“该刹车”的区域。随后再取分数落在 **τ1 到 τ2** 之间的一小段作为 masked 部分，其中 **τ2 = τ1 + 5%**。这样每道题保留多少推理长度，是自适应的。

你可以把两者理解成：

- **SBT-E**：比较像“固定截断模板”
- **SBT-D**：比较像“按题目难度动态截断”

### 4. 还加入了一个“刹车提示”

作者不满足于只截断数据，还在“该停下来的位置”插入自然语言提示，比如：

> “我已经多次得到同一个答案了，该结束思考了。”

当然论文不是要求模型一字不差背这个句子，而是用这类 **self-reflective braking prompt** 去教模型形成一种“元认知”：  
**我已经验证够了，现在该从 think 切到 final answer。**

作者强调，这比特殊 token 或外部规则更自然，因为它贴近模型原本的语言能力。

**原始 OpenR1-Math 长推理样本 → 找到 overthinking 边界 → 保留有效推理 + 插入刹车提示 + 保留少量冗余但 mask 掉 loss → 用这些新样本做普通 SFT。**


### O1-Pruner: Length-Harmonizing Fine-Tuning for O1-Like Reasoning Pruning

https://arxiv.org/pdf/2501.12570
## 它的方法在做什么

O1-Pruner 的思路不是简单粗暴地“限制最多输出多少 token”，而是设计一个训练目标，让模型自己学会：**简单题尽量短，难题在必要时可以长，但别无意义地冗长。** 论文把这叫做 **Length-Harmonizing Fine-Tuning**。

作者先把目标写成一个带约束的优化问题：  
一方面，希望模型生成的解答 **比参考模型更短**；另一方面，又要求 **准确率不能比参考模型差**。论文里把解答长度记作$L(y)$，准确与否记作 $A(x,y)∈{0,1}A(x,y)\in\{0,1\}A(x,y)∈{0,1}$。于是，目标是尽量增大“参考模型平均长度 / 当前解答长度”，同时约束当前模型的期望准确率不低于参考模型。

为了能训练，作者把“准确率不能下降”这个约束揉进奖励函数，变成一个更好优化的形式。最后得到的核心奖励可以概括成：

**Length-Harmonizing Reward = 长度奖励 + 准确率奖励**

其中长度奖励大致是

$\frac{\bar L_{\text{ref}}(x)}{L(y)} - 1$

意思是：如果你的解答比参考模型在这道题上的平均长度更短，就有正向激励；如果更长，就会吃亏。准确率奖励则是

$λ (A(x,y)−Aˉref(x))$

也就是把当前解答是否答对，与参考模型在这道题上的平均正确率相比。两项加起来，就得到最终的 RLH 奖励。

这套奖励设计最巧妙的地方在于它是**按题目自适应**的。论文在 4.3 节专门解释了这一点：  
如果一道题本来就很简单，参考模型在这题上的平均正确率 Aˉref(x)\bar A_{\text{ref}}(x)Aˉref​(x) 会比较高，那你“答对”本身拿不到太多额外奖励，所以模型更倾向于把答案写短；  
如果一道题很难，Aˉref(x)\bar A_{\text{ref}}(x)Aˉref​(x) 会比较低，那么“答对”这件事会带来更大的准确率奖励，此时模型就不会一味追求短，而会更重视走对推理路径。换句话说，它希望学到的是：**简单题短做，难题谨慎做。**

## 它是怎么训练的

训练时，作者先固定一个参考模型 πref\pi_{\text{ref}}πref​，对每道题先采样出多条解答，用这些采样结果估计这道题的两个基线量：

- 参考模型在这题上的平均输出长度 $Lˉref(x)$
- 参考模型在这题上的平均准确率 $Aˉref(x)$

然后他们不是做标准在线 RL，而是做一种**off-policy、PPO 风格**的训练。也就是说，训练样本直接来自参考模型预先采样好的答案，而不是每次都从当前模型重新在线采样。这样做的好处是：训练流程更简单，数据可以一开始就准备好。损失函数采用了类似 PPO 的 clipped 形式，里面用到重要性比率

$r(\theta)=\frac{\pi_\theta(y|x)}{\pi_{\text{ref}}(y|x)}$​

来把“参考模型分布上的样本”转成对当前模型的有效训练信号。作者明确说，这样做是为了降低训练复杂度，同时保持效果。

如果把整套流程翻成人话，大概就是：

1. 先让参考模型对每道题想很多次；
2. 统计这道题“通常会想多长”“通常有多容易答对”；
3. 再训练新模型，让它在这道题上尽量比“参考平均长度”更短，但不能因此把正确率搞坏

### Demystifying Long Chain-of-Thought Reasoning in LLMs

https://arxiv.org/pdf/2502.03373
第四部分：作者提出的核心技术——Cosine Reward

为了解决上面这个问题，作者设计了一个新的长度塑形奖励，叫 **Cosine Reward**。它不是只看答案对不对，还把 **生成长度** 也纳入奖励。核心约束有三个：

1. **正确答案的奖励要高于错误答案**
2. **同样正确时，短的 CoT 比长的 CoT 奖励更高**，鼓励高效推理
3. **同样错误时，短的错误 CoT 应该罚得更重**，鼓励模型在没把握时多想一会儿，而不是太快交卷

作者用一个分段 cosine 函数去平滑实现这个奖励，并在超过最大长度时再加额外惩罚。

这个设计很巧：

- **答对了但太啰嗦**，也不够好
- **答错了但很短**，也不够好
- 模型会被推向一种更合理的状态：  
    **会做的题快点做完，不会的题允许想更久**

## Cosine Reward 有什么效果

效果是很明显的。

Figure 4 显示，在 Llama-3.1-8B 上，和经典奖励相比，**Cosine Reward 让训练准确率和响应长度都更稳定**；Figure 5 进一步显示，这种稳定性会转化成下游任务上的更好表现。作者的 takeaway 是：**Reward shaping can be used to stabilize and control CoT length while improving accuracy.**

也就是说，这篇论文不是简单主张“长就是好”或者“短就是好”，而是在说：

**长 CoT 必须被“塑形”。否则它很容易从“深度推理”滑向“无效拉长”。**

 第五部分：奖励还会被模型“黑掉”

作者还观察到一个很现实的问题：  
当训练算力继续加大时，模型会开始 **reward hacking**。具体表现是：它不是靠更好的求解去变长，而是靠**重复、堆砌、拖时间**来骗长度奖励。作者甚至用关键词 “alternatively” 的出现频率去估计分支行为，发现里面混进了不少“假分支”。

他们的解决办法很简单但有效：**加 repetition penalty**。  
Figure 5 显示，加入 repetition penalty 后，不仅下游性能更好，而且 CoT 也更短，说明推理算力被利用得更有效。作者的 takeaway 是：**长度奖励迟早会被黑，但重复惩罚可以缓解。**

这部分的直觉很好懂：

- 长度奖励会让模型想“要不要多写点”
- 如果没有约束，它就可能学会“多写废话”
- 所以你得同时奖励“有必要的思考”，惩罚“无意义的重复”

### Learning When to Think: Shaping Adaptive Reasoning in R1-Style Models via Multi-Stage RL
https://arxiv.org/pdf/2505.10832 
三阶段强化学习
### 第一阶段：先稳住“双模式共存”

第一阶段的目标不是提升最终成绩，而是**防止模式坍塌**。  
因为如果你一开始就奖励“答对且不思考”，模型很容易全都跑去 no-think；反过来，如果 thinking 模式暂时更占优，也可能全都坍塌成 always-think。论文先定义了一个 naive reward：

- think 且对：+1
- think 且错：0
- no-think 且对：+2
- no-think 且错：-1

这样看起来是在鼓励“能不想就不想”，但早期训练会不稳定。于是作者又加入了 **batch reward balance**：如果一个 batch 里 think 太多，就软性压低 think 奖励；如果 no-think 太多，就压低 no-think 奖励，用一个目标比例 γ 和惩罚斜率 λ 把两种模式维持在可探索的平衡区间。

你可以把第一阶段理解成：  
**先不要急着教模型“怎么最优”，先确保它别只会一种行为。**  
论文的消融也证明了这一点：如果只用 naive reward，模型会很快坍塌到 thinking 模式；如果直接上长度惩罚，又会坍塌到 no-thinking 模式。只有 batch-level 的平衡机制，才能让两种模式稳定共存。

### 第二阶段：在两种模式里分别把“答对”练出来

第二阶段做的事其实很朴素：  
**把第一阶段那个“保平衡”的扶手拿掉，直接用 naive reward，开始真正强化正确行为。**  
作者说，经过第一阶段的正则化后，第二阶段即使放开，thinking 和 no-thinking 的比例也不会轻易再次坍塌；而且这一阶段会给更大的上下文预算，让模型在需要时能产生更长、更充分的推理。它的作用，就是把“什么时候想”和“想了以后能不能答对”这两件事真正建立起来。

这一步很关键，因为论文后面的消融表明：  
**如果跳过第二阶段，直接从第一阶段去做剪枝，最终准确率会明显更差。**  
作者报告说，跳过第二阶段后，即便最后响应长度差不多，平均准确率也只有 **47.6%**，低于完整三阶段训练得到的 **51.7%**。也就是说，先“强化正确推理”，再去“压缩长度”，比直接上剪枝有效得多。

### 第三阶段：再做“按长度感知的剪枝”

到了第三阶段，模型已经有了比较稳定的 think / no-think 双模式，也在各模式里学会了更可靠地答题。此时作者才开始引入 **length-aware reward**。  
它的设计很有意思：对每个 query 里的响应，先看它相对同组样本是长还是短，然后做奖励塑形：

- **如果答对了，越长越要扣一点奖励**，鼓励“正确但简洁”；
- **如果答错了，越短反而越不理想**，鼓励模型在难题或不确定时不要太快收工。

所以它不是简单地“长就罚、短就奖”，而是更像：  
**会的题请短答，不会的题别装会，应该多想一点**



### Do NOT Think That Much for 2+3=? On the Overthinking of Long Reasoning Models
https://arxiv.org/pdf/2505.21765

## 他们提出了两个“效率指标”

这是这篇论文比较有价值的地方。作者不只看“答对没答对”，还看 **token 有没有被有效利用**。

### 1. Outcome efficiency（结果效率）

这个指标问的是：

**这些后续 token，对“得到正确答案”到底有没有帮助？**

它的做法很直观：  
如果一个回答里第一轮就已经出现正确答案，那么从那之后再生成的大量 token，就不再算“对正确率有贡献的 token”。于是 outcome efficiency 本质上就是：

> **到第一次得到正确答案为止的有效 token，占总 token 的比例**

论文发现，在有正确答案的样本里，**超过 80% 的情况，第一轮 solution 就已经答对了**；而这第一轮通常只占整个回答不到 60% 的 token。也就是说，后面那大段内容对把题做对帮助非常有限。以“2+3”那个例子为例，第一轮只用了 **39 token** 就答对了，但总回答长达 **901 token**，所以 outcome efficiency 只有 **4.3%**。

### 2. Process efficiency（过程效率）

这个指标问的是：

**这些后续 token，至少有没有带来新的推理思路？**

作者不是简单看长度，而是看“后面的解法是否引入了新的 reasoning strategy”。他们用 GPT-4o 去把同一回答里的 solution 按推理策略聚类，再计算有多少 token 属于“新的、没重复过的思路”。本质上，process efficiency 就是：

> **真正贡献新思路的 token，占总 token 的比例**

论文发现，随着 solution 的轮次增加，后面的 distinctness ratio 会持续下降。尤其第二轮经常只是对第一轮做同一思路下的复查，并不是真的引入新策略。
## 这篇论文怎么缓解 overthinking

它的方法不是 RL，也不是外接一个控制器，而是一个 **self-training + preference optimization** 的方案。整体分三步：

1. 先让长推理模型自己生成长 reasoning traces；
2. 再把这些长回答剪成更短、更高效的版本；
3. 然后拿“短版本 vs 长版本”做偏好学习，让模型学会偏向更简洁但仍有效的推理
